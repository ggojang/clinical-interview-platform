# Roadmap

## Phase 1 — Foundation and initial vertical slices

- Repository contracts and AI operating rules
- Clinical Memory, Fact, Pattern, Provenance, and Simulation schemas
- Respiratory fact catalog
- Cough interview pattern
- Respiratory red flags
- Synthetic cough patients
- Validator and transparent demo runner
- Primary Care Reason for Encounter catalog
- Fever Fact, Rule, Simulation, and Knowledge Package vertical slice
- Breathing-difficulty Fact, Rule, Simulation, and Knowledge Package vertical slice
- Shared Fact identity across independently compiled RFE packages

Current next Tier 1 slices:

- Chest pain
- Abdominal pain

## Phase 2 — Evaluation depth

- 20 representative interview patterns
- 200 reviewed or review-ready facts
- 100 synthetic simulations
- Contradiction, hallucination, red-flag, coverage, and efficiency metrics
- Multi-model generation with separated provenance

## Phase 3 — Terminology and interoperability

- SNOMED CT concept alignment where licensing permits
- LOINC and FHIR mappings
- Guideline ingestion and evidence tracking
- FHIR export contracts and validation

## Phase 4 — Governed pilot

- Expert review workflow
- Privacy and security controls
- Clinician-facing pilot environment
- Prospective evaluation and continuous knowledge evolution

## Gate rule

A phase does not advance because file count increased. It advances only when its acceptance tests pass.
